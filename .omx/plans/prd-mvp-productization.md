# PRD: Offline MVP Productization

## Decision
Ship the next safe layer for the completed auto-trading bot MVP: user-facing setup docs, deterministic examples, sample reports, and regression coverage for those artifacts. This run must not move toward live trading.

## Goals
1. Make the MVP runnable by a new local user without reading source code.
2. Provide deterministic sample OHLCV data and committed sample report artifacts that show expected output shape.
3. Explain safety boundaries and future live/paper-trading approval gates clearly.
4. Add tests that keep docs/examples in sync with the CLI and preserve safety statements.
5. Preserve the existing no-dependency, offline-only package boundary.

## Non-goals
- No broker, exchange, paper-trading, or market-data adapter implementation.
- No network calls, credentials, `.env`, account management, or order-routing features.
- No investment advice, profit claims, or production trading recommendation.
- No dependency additions.

## Users
- Primary: local developer/operator who wants to run offline validation against CSV fixtures.
- Secondary: future maintainer who needs a clear gate before broker/paper/live extensions.

## User stories
- As a local user, I can install/run tests and execute a sample backtest command from README.
- As a local user, I can inspect a sample CSV and generated Markdown/JSON reports.
- As a maintainer, I can see what must happen before any future paper/live trading work.
- As a reviewer, I can run tests that fail if example reports drift from CLI behavior or lose safety caveats.

## Acceptance criteria
- `README.md` exists and documents install, test, CLI example, CSV schema, report interpretation, and safety boundary.
- `examples/` contains a deterministic local CSV fixture and sample report bundle generated by current CLI.
- Documentation explicitly states this MVP cannot place orders and is not approval for live trading.
- Tests verify the README command shape and committed example reports are reproducible/safe.
- Full verification passes: pytest, ruff, mypy, compileall, tabnanny, static safety scan.

## Constraints and safety invariants
- All examples are local files; commands use local fixture paths and local output directories.
- Reports must keep `live_trading_authorized=false` and the required safety statement.
- Production code must remain free of banned broker/network/credential/order pathways.
